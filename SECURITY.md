# Security policy

## Reporting a vulnerability

If you discover a security vulnerability in this template, **do not open a public issue.**

Instead, report it privately via one of:

- **GitHub Security Advisories** (preferred): https://github.com/navapbc/digital-service-orchestra-nextjs-template/security/advisories/new
- **Email** the Digital Service Orchestra maintainers: opensource@navapbc.com

Please include:
- A description of the vulnerability and its impact.
- Reproduction steps.
- The version (commit SHA) where the issue exists.
- Any suggested mitigation.

We will acknowledge receipt within 5 business days and aim to provide a remediation plan within 30 days.

## Scope

This policy covers the **template repository itself** — the code shipped in this repo. Vulnerabilities in projects scaffolded *from* this template should be reported to those projects' maintainers.

## Coordinated disclosure

We follow a coordinated disclosure model. Please give us a reasonable window to remediate before publicly disclosing.

Thank you for helping keep the Digital Service Orchestra ecosystem secure.
